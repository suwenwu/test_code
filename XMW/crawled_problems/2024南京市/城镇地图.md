# 【2024南京市信息与未来】城镇地图

> 时空限制：1秒 / 128MB

## 题目描述

乐乐管理了⼀个巨⼤的城镇 。城镇可以看作是⼀个 $n$ ⾏ $m$ 列的矩形 ，其中 ，城镇的中⼼位于 第 $x_0$ ⾏ 、第 $y_0$ 列 。城镇中每个格⼦都⽤⼀个字符来表⽰ ，城镇地图的绘制⽅法如下：
- ⾸先在城镇的中⼼画上⼀个加号 +。
- 给穿过城镇中⼼的⼀⾏画上减号 -。(城镇中⼼位置不画)
- 给穿过城镇中⼼的⼀列画上竖线 |。(城镇中⼼位置不画)
- 从城镇中⼼出发 ，分别给斜左上⽅ 、斜右上⽅ 、斜右下⽅ 、斜左下⽅画上 \\、/、\\、/ 字符 ，⼀直绘制直到到达地图边界。
- 以上横竖斜线将城镇分割为⼋个区域 ，按照顺时针⽅向给每个区域分别填上 A-H ⼋个字母

## 输入格式

输⼊第⼀⾏四个整数 $n, m, x_0 , y_0$ ，分别代表城镇区域的⾼度 ($n$ ⾏) 、宽度 ($m$ 列) 和城镇中⼼的位置 ($x_0 , y_0$ ) 。城镇中⼼的位置满⾜ $1 ≤ x_0 ≤ n, 1 ≤ y_0 ≤ m$。

输⼊第⼆⾏⼀个整数 $Q$ ， 代表绘制地图的请求数量 。接下来 $Q$ ⾏ ， 每⾏两个整数 $x, y (1 ≤ x ≤ n, 1 ≤ y ≤ m) $，表⽰⼀个请求 ，绘制出以第 $x$ ⾏第 $y$ 列为中⼼的 $5 × 5$ 地图。

## 输出格式

为每个请求 $x, y$ 输出 $5$ ⾏ ，每⾏ $5$ 个字符 ，为绘制的地图 。超出城镇边界的部分画井号 “#”。

为了更好的视觉效果 ，你可以选择在每个请求之后输出⼀个额外的空⾏ (是否输出额外空行不影响正确性)。

## 输入输出样例

```input
6 15 4 8
2
1 1
2 7
```

```output
#####
#####
##GGG
##GGG
##GGG

#####
\\HH|A
G\\H|A
GG\\|/
---+-\
```

## 数据范围与提示

- 对于 $60\%$ 的数据 ，满⾜ $n, m ≤ 1, 000$。
- 对于 $100\%$ 的数据 ，满⾜ $1 ≤ n, m ≤ 1, 000, 000$。
- 对于 $100\%$ 的数据 ，满⾜ $Q ≤ 1, 000$。
